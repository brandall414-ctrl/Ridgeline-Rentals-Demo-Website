# Ridgeline Rentals — Marketing Website

A two-page, pre-launch marketing site for **Ridgeline Rentals**, a self-serve via
ferrata gear rental business at the Ashley Gorge Via Ferrata trailhead near Vernal,
Utah. This is a demo / pre-launch site built to support BLM Special Recreation
Permit approval and insurance underwriting. It will be replaced with a full
transactional site after approval.

Built with **vanilla HTML5, CSS3, and a small amount of vanilla JavaScript** — no
frameworks, no build step, no dependencies to install.

---

## Pages

| File | Purpose |
|---|---|
| `index.html` | Landing page — hero, what is via ferrata, how the locker works, pro tips, trust, contact, photo gallery |
| `rates.html` | Gear rental rates — route map, climber requirements, pricing, gear protection, documents |

## Editing the copy (no HTML required)

All text lives in the three JSON files in `/content/`. On page load, `js/main.js`
fetches the JSON and injects it into the page. To change wording, edit the JSON and
save — no HTML edits needed.

- **Global things** (business name, phone, email, location, nav links, copyright):
  `content/site.json`
- **Landing page copy** (hero, steps, tips, etc.): `content/landing.json`
- **Rates page copy** (pricing, requirements, fine print): `content/rates.json`

How it works under the hood: HTML elements carry a `data-content-key` attribute
with a dot path (e.g. `data-content-key="hero.headline"`), and the JS looks that
path up in the JSON. Repeating blocks (steps, tips, pricing cards, requirements)
are generated by cloning `<template>` elements from the arrays in the JSON. If a
value is missing, the default copy already in the HTML stays in place, so the page
never breaks.

> **Note:** Because the pages load the JSON with `fetch()`, they must be served over
> **http://**, not opened as a `file://` path. See "Running locally" below. When
> deployed to Netlify this is automatic.

## Running locally

You need any static file server. With Python (already on most Macs):

```bash
cd ridgeline-rentals-web
python3 -m http.server 4173
# then open http://localhost:4173
```

Or with Node, if you prefer: `npx serve` .

## Design system

Defined as CSS custom properties at the top of `css/styles.css`.

- **Colors:** warm cream backgrounds, terracotta canyon orange (primary), golden
  sunset (secondary), deep forest green (CTAs/links), near-black charcoal text.
- **Type:** `Bitter` (slab serif) for headings, `Inter` for body — loaded from
  Google Fonts.
- **Motion:** sections and cards fade/slide in on scroll (IntersectionObserver),
  the hero image has a slow Ken-Burns drift, and the sticky header turns from
  transparent to solid cream as you scroll past the hero. All motion is disabled
  automatically for visitors who prefer reduced motion.
- **Canyon motifs:** the layered ridgelines in the hero, section dividers, the
  CTA band, and the footer are hand-drawn inline SVG — no external image files.

## What this site intentionally does NOT do

Per the project brief, this is an informational pre-launch site:

- No booking, payment, or Stripe integration.
- No forms that submit anywhere — the "Get Notified" buttons are `mailto:` links.
- No "Book Now" language — everything is "Get Notified" / informational.
- The production domain will be **ridgelinerentals.org** (not `.com`).

## Deployment

The site is static, so it deploys as-is. See the push-to-GitHub and Netlify steps
that were provided alongside this build. In short: push to a GitHub repo named
`ridgeline-rentals-web`, connect that repo in Netlify, and deploy with **no build
command** and a publish directory of `.` (the repo root).
